# Pairing Code Integration Fix

## Problem
Pairing code was not displayed in frontend because of status mismatch between backend and frontend.

## Root Cause
- **Backend** saved pairing code with status `'waiting_pairing'` (pairing-handler-stable.js:141)
- **Frontend** only checked for pairing code when status is `'connecting'` (Devices.tsx:291)
- Result: Frontend polling never detected the pairing code

## Solution
Changed backend to use status `'connecting'` instead of `'waiting_pairing'` to match frontend expectation.
